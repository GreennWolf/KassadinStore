Introducción

Este documento resume las mejoras implementadas en el proyecto Kassadin Store, una plataforma de venta de productos y servicios para League of Legends. Las mejoras se han centrado principalmente en el sistema de EloBoost, la gestión de pagos y la interfaz de administración.

Corrección de Encriptación

Problema

El sistema de encriptación para las credenciales de cuenta en el servicio de EloBoost fallaba con el error ERR_CRYPTO_INVALID_KEYLEN, debido a que la clave de encriptación no tenía la longitud requerida para el algoritmo AES-256-CBC.

Solución

Se implementó una función getFixedLengthKey() que garantiza que la clave tenga exactamente 32 bytes:

  • Si la clave es más corta, la rellena con ceros
  • Si la clave es más larga, la trunca
// Asegurarnos que la clave tenga exactamente 32 bytes (256 bits) para AES-256
const getFixedLengthKey = (key) => {
    // Si la clave es menor a 32 bytes, rellenarla
    if (key.length < 32) {
        return key.padEnd(32, '0');
    }
    // Si la clave es mayor a 32 bytes, truncarla
    if (key.length > 32) {
        return key.substring(0, 32);
    }
    return key;
};

Además, se mejoró el manejo de errores en las funciones de encriptación y desencriptación para proporcionar mensajes más claros en caso de fallos.

Mejoras en el Sistema de EloBoost

Visualización de Detalles de Orden

Se actualizó el componente EloBoostOrderDetailsModal para mostrar información más completa:

  1. Estructura de pestañas mejorada:
    • Detalles: Información básica del boost solicitado
    • Cuenta: Credenciales de la cuenta del cliente
    • Pago: Detalles y comprobante del pago
    • Notas: Observaciones internas
  2. Visualización de credenciales segura:
    • Las credenciales se muestran en un formato seguro
    • Advertencia sobre la confidencialidad de los datos

Proceso de Checkout

Se optimizó el flujo de checkout para evitar la creación prematura de órdenes:

  1. Almacenamiento en sessionStorage:
    • Los datos del boost se guardan temporalmente durante el proceso
    • La orden se crea solo cuando todos los datos están completos
  2. Validación mejorada:
    • Validación paso a paso de la información
    • Mensajes de error específicos para cada campo

Visualización de Pagos

Comprobantes de Pago

Se implementó un sistema completo para la visualización de comprobantes:

  1. Modal de imagen:
    • Vista previa del comprobante de pago
    • Opción de ampliar la imagen para mejor visualización
  2. Estado del pago:
    • Indicador visual claro del estado (Pagado/Pendiente)
    • Detalles del método de pago utilizado
{order.payment?.receiptImg && (
  <div className="mt-4">

        onClick={() => setIsImageModalOpen(true)}
      />
    </div>
  </div>
)}

Mejoras en la Interfaz de Administración

Tabla de Órdenes

Se actualizó la visualización de órdenes con nuevas funcionalidades:

  1. Columna de estado de pago:
    • Badges visuales para distinguir rápidamente órdenes pagadas y pendientes
    • Acciones contextuales según el estado
  2. Sistema de filtrado ampliado:
    • Filtro por estado de la orden
    • Filtro por estado del pago (Pagado/Pendiente)
    • Búsqueda por ID o usuario
<Select
  value={filters.paymentStatus}
  onValueChange={(value) => handleFilterChange('paymentStatus', value)}
>
  <SelectTrigger>
    <SelectValue placeholder="Filtrar por pago" />
  </SelectTrigger>
  <SelectContent>
    <SelectItem value="">Todos los pagos</SelectItem>
    <SelectItem value="paid">Pagados</SelectItem>
    <SelectItem value="pending">Pendientes de pago</SelectItem>
  </SelectContent>
</Select>

Acciones Administrativas

Se implementaron botones de acción para agilizar la gestión:

  1. Botones de transición de estado:
    • Iniciar proceso para órdenes confirmadas
    • Marcar como completado para órdenes finalizadas
  2. Visualización de comprobantes:
    • Acceso rápido a los comprobantes de pago
    • Botón específico para verificar pagos pendientes

Conclusión

Estas mejoras optimizan significativamente el sistema de EloBoost, especialmente en lo relacionado con la seguridad de las credenciales, la gestión de pagos y la experiencia administrativa. El sistema ahora proporciona un flujo más lógico para el proceso de checkout y facilita a los administradores la verificación de pagos y la gestión de órdenes.
